# VKampus Security Module (Open Source Core)

> 🛡️ **Мы за прозрачность.** Этот репозиторий содержит ядро безопасности и криптографии платформы ВКампус.

Архитектура ВКампус выстроена вокруг принципа "Privacy by Design" (приватность по умолчанию). В качестве доказательства того, что мы не собираем личные переписки, контакты или номера телефонов, мы выкладываем в открытый доступ код, который используется на нашем бэкенде для аутентификации пользователей через Telegram.

## Что находится в этом репозитории?

- **`lib/auth-validator.ts`** — Проверка цифровых подписей (HMAC-SHA256) от API Telegram. Гарантирует, что запрос не подделан и действительно отправлен из официального клиента.
- **`lib/crypto-utils.ts`** — Алгоритмы анонимизации (необратимое криптографическое хеширование Telegram ID). Мы не храним ваш реальный `telegram_id` в открытом виде.

## Почему не весь проект?
Целиком ВКампус остается закрытым (проприетарным), чтобы защитить уникальную бизнес-логику, алгоритмы ранжирования постов, архитектуру базы данных и предотвратить создание недобросовестных клонов. Однако мы считаем, что всё, что касается **обработки персональных данных и алгоритмов шифрования**, должно быть публичным и проверяемым (open-source). 

Мы строго следуем [Принципу Керкгоффса](https://ru.wikipedia.org/wiki/Принцип_Керкгоффса): система должна оставаться безопасной, даже если злоумышленнику известен исходный код алгоритмов ее работы, за исключением секретных ключей.

---

### Аудит безопасности
Ключевые механизмы, которые мы используем для защиты студентов:

1. **Нет паролей:** ВКампус вообще не использует пароли, которые можно украсть или перехватить. Авторизация происходит строго по криптографическому токену сессии Telegram (WebAppData).
2. **Нулевой доступ к профилю:** Механизм `validateInitData` технически запрашивает у Telegram только базовую визитку (id, имя). У нас просто нет ключей доступа к вашим чатам, подпискам или списку контактов.
3. **Хеширование личности:** Даже если злоумышленник получит полный дамп нашей базы данных "users", он увидит лишь хеш-суммы формата `a3f8b...`. Сопоставить этот хеш с реальным профилем Telegram без нашего секретного ключа (соли) невозможно.

Если вы являетесь специалистом по информационной безопасности, мы всячески приветствуем аудит этого кода.
